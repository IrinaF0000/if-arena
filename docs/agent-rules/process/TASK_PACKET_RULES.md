# Task Packet Rules

A task packet must be short, scoped, and executable.

Required sections:

- goal and non-goals if needed;
- allowed and forbidden files;
- required quality gates;
- implementation steps;
- required tests, including the Test Impact Matrix from `docs/agent-rules/quality/TESTING.md`;
- acceptance criteria;
- required reviews;
- rollback note.

Do not duplicate large rule blocks from canonical docs. Link to `QUALITY_GATES.md`, `TESTING.md`, and `SEQUENTIAL_AGENT_PIPELINE.md` instead.

Task packets live in `docs/agent-tasks/`. Run notes live in `docs/agent-runs/`.
